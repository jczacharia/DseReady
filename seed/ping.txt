Introduction
Introduction 
Active Directory Domain Services (AD DS), provides the methods for storing directory data and making this data available to network users and administrators. AD DS stores information about user accounts, such as names, passwords, phone numbers, and so on. AD DS also provides a way for authorized users on the same network to access this information. AD DS includes:

Schema that defines the classes of objects and attributes contained in the directory, the constraints and limits on instances of these objects, and the format of their names.
Global Catalog that contains information about every object in the directory. Users and administrators can use the catalog to find directory information regardless of the directory domain that actually contains the data.
Query and index mechanism, so that objects and their properties can be published and found by network users or applications.
Replication service that distributes directory data across a network.
Sign-in authentication and access control to objects in the directory.
Group Policy enables configuration and settings management of user and computer settings on computers running Windows Server and Windows Client operating systems.
Active Directory Domain Services exists in all environments at PNC under the WDS mnemonic.

Introduction
Information stored in Active Directory can be accessed via standard LDAP tools and methodologies. Most commonly the connections are made via Secure LDAP (SLDAP) or LDAP over SSL.

Active Directory uses the Lightweight Directory Access Protocol (LDAP) as its access protocol. LDAP search requests can be sent and received by Active Directory on port 389 (the default LDAP access port) and port 3268 (the default global catalog port). LDAP traffic that uses the Secure Sockets Layer (SSL) authentication protocol accesses ports 636 and 3269, respectively.

Bind account
The directory does not allow anonymous look ups, therefore an account is required to query the directory. While a personal account can be used temporarily for testing, a permanent service account should be configured within the application's LDAP configuration.
Obtain a bind account by submitting a Powerlink Windows Non Human Account Access request.
When setting up a production account to query user objects and/or entitlements be sure to select the "PNCNT (pncbank.com)" domain radio button on the form.

LDAP Policy
To protect the directory hosts from trying to process run-away queries the following two policy settings limit the quantity of results that can be returned at a time:
MaxPageSize - controls the number of object that are returned in a single search result. Value=1,000.
MaxValRange - controls the number of values that are returned for an attribute of an object. Value=1,500.
Troubleshooting
Basic Questions:
What specific domain controller / IP address is the application or host trying to use?
the application should be pointed to the domain name, i.e. "pncbank.com" and NOT individual domain controller hosts.
the domain name is served by multiple domain controllers at multiple IP addresses, "multi-homed".
Try pinging the domain name from the host performing the query and note which IP address is resolved.
the DNS cache may cause the ping to resolve the name to the same IP address but it is not guaranteed.
refresh the host's DNS cache to resolve to a different domain controller. DNS assigns the various domain controller addresses in round-robin fashion so this may or may not provide a different address.
attempt to recycle the service creating the LDAP queries.
What is the error code being returned by the application?
Inability to connect to LDAP
Inability to authenticate to the directory via the bind account
LDAP query not returning (correct) results, mistake in the filter or base DN syntax
What service / non-human / bind account is it using to authenticate to the directory?
Ensure the service account has not been locked out.
Is an error returning right away or is it timing-out?
Is it connecting via LDAP, SLDAP (LDAP over SSL), or ADSI?
If LDAP, how are the query string and parameters configured?
If SLDAP, check certificate and test SLDAP - Certificate Check
Does it utilize the global catalog?
What authentication mechanism is it using? Basic, Kerberos, NTLM?
From what server(s) is/are the authentications coming from?
Is the interaction something we wrote or vendor provided?
How often does the query run? Every time a user logs in? On a schedule?
Is the query failing every time or intermittently?

External Cross-Reference
An external cross-reference is a cross-reference object that can be created manually to provide the location of an object that is not stored in the forest. If your LDAP clients submit operations for an external portion of the global LDAP namespace against servers in your forest, and you want your forest's servers to refer the client to the correct location, you can create a cross-reference object for that directory in the Partitions container. There are two ways that external cross-references are used:

To reference external directories by their disjoint directory name (a name that is not contiguous with the name of this directory tree). In this case, when you create the cross-reference, you create a reference to a location that is not a child of any object in the directory.
To reference external directories by a name that is within the Active Directory namespace (a name that is contiguous with the name of this directory tree). In this case, when you create the cross-reference, you create a reference to a location that is a child of a real directory object.
http://technet.microsoft.com/en-us/library/cc978014.aspx

Paging and Ranging
LDAP servers throttle results to protect themselves from long running, recursive, or expensive queries. It also helps protect against DoS (Denial of Service) attacks.

Page / PagingManages the number of objects returned by a query, throttled to 1,000. See #MaxPageSize above.Range / RangingManages the number of values returned by a multi-value attribute, throttled to 1,499 results.

Range Looping Example in PowerShell
$adfind = "C:\Users\pt43033\Downloads\adfind.exe"
$dn = "CN=GSGu_CLP_CIBRMOnboarding,OU=OUg_Applications,OU=OUc_AccessGroups,DC=pncbank,DC=com"
# $dn = "CN=Domain Admins,OU=OUg_LegacyGroups,OU=OUc_AccessGroups,DC=pncbank,DC=com"

$start = 0
$step = 499
$total = 0
$range = ("member;range=" + ($start) + "-" + ($start+$step))

[array]$results = &$adfind "-noautoranging" "-qlist" "-b" $dn $range
$count = $results.Count
$total += $count
$times = 1
Write-Host ("{0,25}" -f $range + "  Times: " + "{0:d3}" -f $times + "  Total: " + "{0:d4}" -f $total)

while ($count -gt ($step-1))
  {  # while
    $times++
    $range = ("member;range=" + ($total) + "-" + ($total+$step))
    [array]$results = &$adfind "-noautoranging" "-qlist" "-b" $dn $range
    $count = $results.Count
    $total += $count
    Write-Host ("{0,25}" -f $range + "  Times: " + "{0:d3}" -f $times + "  Total: " + "{0:d4}" -f $total)
  }  # while
